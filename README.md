## Задание 1 — `@Invoke`
### Цель задания
Создать аннотацию, которая помечает методы для автоматического вызова во время выполнения программы.

### Требования
- Аннотация применяется **только к методам**
- Аннотация доступна во время исполнения программы
- Аннотация не имеет свойств
- Необходимо автоматически вызывать все методы с этой аннотацией

### Реализация
**Пакет:** `task1`

- **`Invoke`**  
  Аннотация, которой помечаются методы, подлежащие автоматическому вызову.

- **`InvokeExample`**  
  Класс с методом `execute()`, помеченным аннотацией `@Invoke`.  
  Метод изменяет состояние объекта (устанавливает флаг `executed`).

- **`InvokeProcessor`**  
  Обработчик, который с помощью Reflection API находит все методы с аннотацией `@Invoke` и вызывает их.

## Задание 2 — `@Default`
### Цель задания
Создать аннотацию для задания класса по умолчанию.

### Требования
- Аннотация применяется к **классам или полям**
- Доступна во время выполнения программы
- Имеет обязательное свойство `value` типа `Class`

### Реализация
**Пакет:** `task2`

- **`Default`**  
  Аннотация, хранящая класс по умолчанию.

- **`DefaultExample`**  
  Класс, помеченный аннотацией `@Default(String.class)`.  
  Содержит метод `printDefault()`, который выводит имя класса по умолчанию.

## Задание 3 — `@ToString`
### Цель задания
Создать аннотацию для управления формированием строкового представления объекта.

### Требования
- Аннотация применяется к **классам и полям**
- Доступна во время выполнения программы
- Имеет необязательное свойство `value` со значениями `YES` и `NO`
- Значение по умолчанию — `YES`

### Реализация
**Пакет:** `task3`

- **`Mode`**  
  Перечисление с двумя значениями: `YES` и `NO`.

- **`ToString`**  
  Аннотация, определяющая, нужно ли включать поле в строковое представление.

- **`ToStringExample`**  
  Класс с методом `buildString()`, который формирует строку, включая только те поля, для которых разрешён вывод.

## Задание 4 — `@Validate`
### Цель задания
Создать аннотацию для указания списка классов, используемых для валидации.

### Требования
- Аннотация применяется к **классам и аннотациям**
- Доступна во время выполнения программы
- Имеет обязательное свойство `value` типа `Class[]`

### Реализация
**Пакет:** `task4`

- **`Validate`**  
  Аннотация, содержащая массив классов для проверки.

- **`ValidateExample`**  
  Класс, помеченный аннотацией `@Validate` с указанием нескольких классов.

- **`ValidateProcessor`**  
  Обработчик, который извлекает список классов из аннотации.  
  Если массив пустой — выбрасывает `IllegalArgumentException`.

## Задание 5 — `@Two`

### Требования

- Доступна во время выполнения программы
- Имеет два обязательных свойства:

  - `second` типа `int`

### Реализация
**Пакет:** `task5`

- **`Two`**  
  Аннотация с двумя параметрами: строковым и числовым.

- **`TwoExample`**  
  Класс, помеченный аннотацией `@Two`.  
  Метод `printValues()` выводит значения параметров аннотации.
